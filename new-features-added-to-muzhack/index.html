New Features Added to MuzHack

04 August 2016

Two major new features have been introduced in MuzHack: Dedicated bills of materials (BOMs) as part of build instructions (for now only available to GitHub based projects) and PDF versions of build instructions.

Dedicated BOMs

The benefits that dedicated BOMs bring are twofold:

  1. Writing BOMs has become drastically simpler for project authors, since they can simply write their BOMs as concise YAML files, which is much more user friendly than writing the same as Markdown tables. It means that BOMs are also semantically encoded, rather than just baked into the instructions document as regular text. See for example the one for Befaco Rampage, which is quite involved.
  2. BOMs are presented separately from the instructions themselves, and the containing section can be minimized in case the user wants to see just the instructions themselves.

Build Instructions as PDF

PDF documents are now auto generated for build instructions. Just click the corresponding link inside the Instructions tab, for example Befaco Rampage, and you'll download a document ready for printing a paper copy to have by your side as you're actually building the thing. Much more practical than having to check the computer...

Unfortunately, the PDF documents have some ways to go when it comes to visual styling. Hopefully this will get better in the future. Remember though that MuzHack is open source after all, and contributions to improve f.ex. the PDF functionality would be more than welcome!

Arve Knudsen
